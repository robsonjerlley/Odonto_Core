lógica de negócio por método
+ getAdsRoi(channel, period, userId) AdsRoiResultDTO 1. checkOrThrow(userId, ANALYTICS, READ) — apenas ADMIN/MANAGER 2. customers = customerService.findByChannel(channel) 3. customerIds = extrair IDs dos customers 4. winTickets = leadTicketService.findByStatus(WIN) → filtrar em memória pelos customerIds 5. ticketIds = extrair IDs dos winTickets filtrados 6. totalRevenue = dealService.sumRevenueByTicketIds(ticketIds) 7. totalInvestment = adsInvestmentRepository.sumAmountByChannelAndPeriod(channel, from, to) 8. roiMultiplier = totalRevenue ÷ totalInvestment (BigDecimal, HALF_UP, escala 2) — retorna ZERO se investment = 0 9. return new AdsRoiResultDTO(channel, totalInvestment, totalRevenue, roiMultiplier, customers.size(), winTickets.size())
+ getConversionByStage(period, sector, userId) StageConversionResultDTO 1. checkOrThrow — MANAGER vê só o próprio setor 2. tickets no período: createdAt entre from e to, filtrado por sector se informado 3. captureCount = total de tickets no período 4. scheduledCount = tickets com status SCHEDULED ou posterior 5. dealCreatedCount = tickets com Deal associado 6. closedCount = tickets WIN 7. leadsConversionPct = scheduledCount ÷ captureCount × 100 8. evaluationConversionPct = dealCreatedCount ÷ scheduledCount × 100 9. commercialConversionPct = closedCount ÷ dealCreatedCount × 100
+ getDropoffBySector(period, userId) List<SectorDropoffResultDTO> Para cada Sector: entry = tickets que tiveram currentSector=X no período · loss = tickets que saíram como LOSS naquele setor · dropoffPct = loss ÷ entry × 100
+ getUserPerformance(targetUserId, period, userId) UserPerformanceResultDTO 1. resolver targetUser via userRepository → obter sector e role 2. scope: OWN (userId == targetUserId) ou MANAGER/ADMIN (qualquer um) 3. despachar pela sector do targetUser: ATTENDANT → totalAssigned = tickets assignedTo=targetUserId · totalConverted = chegaram a SCHEDULED · avgTicketValue = 0 EVALUATOR → totalAssigned = tickets IN_EVALUATION assignados · totalConverted = deals criados por targetUserId · avgTicketValue = média deal.totalValue COMMERCIAL → totalAssigned = deals recebidos no período · totalConverted = deals WIN closedBy=targetUserId · avgTicketValue = média deal.finalValue · desconto médio = média deal.discountPct 4. conversionPct = totalConverted ÷ totalAssigned × 100 (BigDecimal HALF_UP) 5. bonusApurado = getBonusApurado(targetUserId, periodRef derivado de period, userId)
+ getBonusApurado(targetUserId, periodRef, userId) BigDecimal 1. resolver User via userRepository para obter role e sector 2. config = bonusConfigRepository.findByRoleAndSectorAndPeriodRef(role, sector, periodRef) 3. metricValue = calcular a métrica correspondente ao config.metricKey (ex: "scheduledCount", "closedCount") 4. return metricValue × config.bonusPct ÷ 100 (BigDecimal HALF_UP)
+ getGlobalDashboard(period, userId) GlobalDashboardResultDTO 1. checkOrThrow(userId, ANALYTICS, READ) — userId não filtra dado, serve só pra checar permissão (apenas ADMIN/MANAGER) 2. agrega: getAdsRoi para cada AdsChannel · getConversionByStage sem filtro de setor · getDropoffBySector · top performers por setor
Regra cross-db — AnalyticsServiceImpl lê identity_db (UserRepository) e crm_db (todos os outros) via repositórios injetados pelo Spring. Nunca use @ManyToOne entre entidades de bancos diferentes. FKs para User em crm_db são UUID simples — resolva o User chamando userRepository.findById(uuid) dentro da impl quando precisar de nome ou role.
