Decisões arquiteturais refletidas nos testes
ADRs com cobertura de teste
ADR-003 — ContactLog imutável: testes de DELETE foram removidos. Suíte cobre apenas create, findById e search. Sem update/delete na interface.
ADR-004 — checkOrThrow em métodos críticos: todos os testes de permissão usam doThrow + checkOrThrow, não canAccess(). DealServiceTest, ConfigServiceTest e ContactLogServiceTest cobrem o caminho de acesso negado.
ADR-006 — Customer PII anonimizado: testDeleteCustomerSuccess verifica que anonymize() preenche "CLIENTE ANONIMIZADO" e isAnonymized=true, e que deleteById nunca é chamado.
ADR-007 — RecycleConfig global (sector removido do DTO): setRecycleConfig_success usa RecycleConfigRequestDTO(afterDays) sem sector. Adicionados 6 testes GET para getRecycle, getBonusConfigs e getAdsInvestments.
User Story PPR — cobertura

LeadTicketServiceTest — 5 testes dedicados
5 testes
WIN → POST_PROCEDURE
applyPostProcedure: seta procedurePerformedAt, muda currentSector para LEADS. Role necessário: USER_ATTENDANT / USER_LEADS / ADM_LEADS / ADM_SYSTEM.
POST_PROCEDURE → SCHEDULED (sucesso)
applyScheduledReturn: returnScheduledAt futuro + CPF válido no customer. Seta EVALUATOR e scheduledAt.
POST_PROCEDURE → SCHEDULED (sem data)
returnScheduledAt = null → IllegalStateException em applyScheduledReturn.
POST_PROCEDURE → LOSS (sucesso)
applyLoss com lossReason válido. Seta closedAt. Role: USER_LEADS / ADM_LEADS / ADM_SYSTEM.
POST_PROCEDURE → LOSS (sem motivo)
lossReason = null → IllegalStateException em applyLoss. Nunca salva.
