Sobre o prefixo I nas interfaces: a convenção IUserService vem do C# e é incomum no ecossistema Spring/Java. O padrão Java é UserService para a interface e UserServiceImpl para a implementação — sem prefixo. Como você já tem os *Impl, basta remover o I das interfaces. O Spring injeta pela interface normalmente: @Autowired UserService userService.

Sobre Clinic e ClinicController: não há correspondência no modelo projetado. Se a clínica é uma entidade de configuração (nome, CNPJ, dados da clínica), ela pode entrar como ClinicConfig dentro de commercial/domain. Se não for necessária no MVP, pode ser removida agora e adicionada depois.
